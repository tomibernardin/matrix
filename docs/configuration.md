# Configuration reference

Every variable in `.env`. The file is generated by `setup.sh` from
[`.env.example`](../.env.example) on first run, written mode `600`, and **never
committed**. The `env-drift` CI job guarantees this list stays in sync with
`compose.yml`.

Legend: **auto** = filled by `setup.sh`; **you** = you must set it; **default** =
has a working default you rarely change.

## Host identity

| Variable | Source | Description |
| --- | --- | --- |
| `PUID` | auto | User ID the LinuxServer containers run as. Detected from your user. **Not** `UID` — that's a bash readonly built-in and resolves to 0 under `sudo`. |
| `PGID` | auto | Group ID, same idea. |
| `DOCKER_GID` | auto | Host `docker` group GID. Homepage runs as `PUID:PGID` but joins this group to read `/var/run/docker.sock` for service discovery. Detected via `getent group docker`. |
| `TZ` | auto | Timezone (e.g. `America/Argentina/Buenos_Aires`). Detected from the host; falls back to `UTC`. |

## Paths

| Variable | Source | Description |
| --- | --- | --- |
| `DOCKER_MAIN_ROUTE` | auto | Root for all bind mounts. Defaults to `${HOME}/docker`. Everything below is derived from it. |
| `BACKUP_DIR` | auto | Where `backup.sh` writes nightly archives. Defaults to `${HOME}/matrix-backups`. **Keep on a different disk** than `DOCKER_MAIN_ROUTE`. |
| `*_CONFIG`, `*_DATA`, `*_WORK`, `*_CONF`, `PLEX_MEDIA`, `PLEX_TEMP`, `TRANSMISSION_*`, `NGINXPM_*`, `ADGUARD_*`, `GRAFANA_PROVISIONING` | default | Per-service sub-paths under `DOCKER_MAIN_ROUTE`. Change only if you want a service's data somewhere custom. |

Two of the FileBrowser paths point at **files**, not directories
(`FILEBROWSER_CONFIG=.../settings.json`, `FILEBROWSER_DATABASE=.../filebrowser.db`);
`setup.sh` seeds them as file-stubs so Docker doesn't create them as directories.

## Dashboard (Homepage)

| Variable | Source | Description |
| --- | --- | --- |
| `HOMEPAGE_HOST` | you | Hostname/IP used as the base for tile links — the address **you** type in the browser. Defaults to `localhost`; set to your LAN IP (e.g. `192.168.1.50`) or FQDN so tiles work from other devices. |
| `HOMEPAGE_ALLOWED_HOSTS` | default | Comma-separated Host headers Homepage accepts. `*` = any. |

## Secrets — generated

`setup.sh` writes strong random values for these on first run. Inspect with
`grep -E '_PASS|_PASSWORD|_USER' .env`.

| Variable | Description |
| --- | --- |
| `TRANSMISSION_USER` | Set to `admin`. |
| `TRANSMISSION_PASS` | Random. Transmission RPC password. |
| `GRAFANA_ADMIN_PASSWORD` | Random. Grafana `admin` password (rotate later in the UI). |

## Secrets — you fill

| Variable | Description |
| --- | --- |
| `PLEX_CLAIM` | One-time Plex claim token from <https://www.plex.tv/claim/> (valid ~4 min). Optional — you can claim from the UI instead. |

AdGuard's admin password is **not** an env var — it's set during the first-run
wizard at `http://<host>:3000`.

## Homepage widget credentials — you fill (optional)

These power the live-data widgets on the dashboard. Tiles render without them;
only the metric panels stay blank. Harvest them during
[first-run setup](user-guide.md) and paste them in, then `docker compose up -d`.

| Variable | Where to get it |
| --- | --- |
| `SONARR_API_KEY` | Sonarr → Settings → General → API Key |
| `RADARR_API_KEY` | Radarr → Settings → General → API Key |
| `PROWLARR_API_KEY` | Prowlarr → Settings → General → API Key |
| `BAZARR_API_KEY` | Bazarr → Settings → General → API Key |
| `OVERSEERR_API_KEY` | Overseerr → Settings → General → API Key |
| `PLEX_TOKEN` | From a signed-in Plex session (item → Get Info → View XML → `X-Plex-Token`) |
| `ADGUARD_USERNAME` / `ADGUARD_PASSWORD` | The admin creds you set in the AdGuard wizard |
| `NPM_USERNAME` / `NPM_PASSWORD` | Your Nginx Proxy Manager login |

## Changing configuration safely

1. Edit `.env` (or, for repo-owned service configs, edit them in the repo — see
   [Operations](operations.md#editing-repo-owned-configs)).
2. Validate: `cp .env.example .env.bak 2>/dev/null; docker compose config --quiet`.
3. Apply: `docker compose up -d`.

If you add a new `${VAR}` to `compose.yml`, you **must** also add it to
`.env.example` or CI's `env-drift` job fails — that's the guard keeping this
reference accurate.
